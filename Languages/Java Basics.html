Getting Started

Development Environment Setup

Java requires two things: the JDK (Java Development Kit) to compile/run code, and an IDE like IntelliJ IDEA to write it comfortably. The JDK includes the compiler (javac) and the JVM (Java Virtual Machine).

JDK — Java Development Kit

Download from oracle.com/java or use OpenJDK. Gives you the compiler and runtime. Recommended: JDK 17 (LTS) or JDK 21 (LTS).

IntelliJ IDEA — IDE

The most popular Java IDE. The Community Edition is free and has everything you need for learning. Download from jetbrains.com.

Anatomy of a Java Program

Every Java program lives inside a class. Execution begins at the main method. Java is case-sensitive and every statement ends with a semicolon ;.

HelloWorld.java
// File must be named: HelloWorld.java
public class HelloWorld {

    // main method — entry point of every Java application
    public static void main(String[] args) {
        System.out.println("Hello, World!");   // prints with newline
        System.out.print("No newline here");    // prints without newline
        System.out.printf("Name: %s, Age: %d%n", "Alice", 30); // formatted
    }
}
Output
Hello, World!
No newline hereAlice, Age: 30
Naming RuleThe file name must exactly match the public class name, including capitalization. HelloWorld.java must contain public class HelloWorld.

How Java Code Gets Executed

Java is unique — it compiles to bytecode (not machine code), which then runs on the JVM. This is why Java is platform-independent: "Write Once, Run Anywhere."

Terminal — Compile & Run
# Step 1: Compile .java → .class (bytecode)
$ javac HelloWorld.java

# Step 2: Run the bytecode on the JVM
$ java HelloWorld

# Output:
Hello, World!

Runjava (JVM)HelloWorld.classProgram executes on your OS

Types & Variables

Variables

A variable is a named container for a value. In Java, you must declare the type before using a variable. Use var (Java 10+) to let the compiler infer the type.

Java — Variable Declaration
// Syntax: type variableName = value;
int age = 30;
double price = 9.99;
boolean isActive = true;
String name = "Alice";

// var — type inferred by compiler (Java 10+)
var city = "New York";   // inferred as String
var count = 42;           // inferred as int

// Multiple variables of same type
int x = 1, y = 2, z = 3;
Naming ConventionUse camelCase for variables: firstName, totalAmount. Use ALL_CAPS for constants: MAX_SIZE. Class names use PascalCase: BankAccount.

Primitive Types

Java has 8 primitive types. They store simple values directly in memory (on the stack), making them fast and memory-efficient.

Strings

String is a reference type, but it behaves like a primitive in many ways. Strings are immutable — once created, they cannot be changed.

Java — String Methods
String msg = "  Hello, Java World!  ";

System.out.println(msg.length());          // 22
System.out.println(msg.trim());            // "Hello, Java World!"
System.out.println(msg.toUpperCase());     // "  HELLO, JAVA WORLD!  "
System.out.println(msg.contains("Java"));  // true
System.out.println(msg.replace("Java", "Python")); // replaces text

// String comparison — NEVER use == for content!
String a = "hello";
String b = "hello";
System.out.println(a.equals(b));           // true (compare content)
System.out.println(a.equalsIgnoreCase("HELLO")); // true
Never use == to compare Strings!== compares memory addresses, not content. Always use .equals() for string comparison.

Arrays

An array stores a fixed-size sequence of elements of the same type. Once created, the size cannot change. Array indices start at 0.

Java — Arrays
int[] scores = { 85, 90, 78, 92, 88 };

System.out.println(scores[0]);          // 85 (first)
System.out.println(scores.length);      // 5

// Sort an array
java.util.Arrays.sort(scores);
System.out.println(java.util.Arrays.toString(scores)); // [85, 88, 90, 92, 95]

Type Casting

Widening (automatic) goes from smaller to larger type. Narrowing (manual) goes from larger to smaller and may lose data.

Widening — Automatic
int i = 42;
double d = i;   // int → double, safe
System.out.println(d); // 42.0
Narrowing — Manual Cast
double pi = 3.99;
int n = (int) pi;   // truncates to 3
System.out.println(n); // 3 (not 4!)

If / Else If / Else

Java — If-Else
int temperature = 28;

if (temperature > 30) {
    System.out.println("Hot day!");
} else if (temperature > 20) {
    System.out.println("Nice weather");     // ← executes this
} else {
    System.out.println("Bundle up!");
}

// Ternary operator
int age = 20;
String status = (age >= 18) ? "Adult" : "Minor";
System.out.println(status);  // Adult

Switch Statements

Traditional Switch
int day = 3;
switch (day) {
    case 1: System.out.println("Monday"); break;
    case 2: System.out.println("Tuesday"); break;
    case 3: System.out.println("Wednesday"); break;
    default: System.out.println("Other");
}
Switch Expression (Java 14+)
String dayName = switch (day) {
    case 1 -> "Monday";
    case 2 -> "Tuesday";
    case 3 -> "Wednesday";
    default -> "Weekend";
};
System.out.println(dayName); // Wednesday

For, While & For-Each Loops

Java — Loops
// For loop
for (int i = 1; i <= 5; i++) {
    System.out.println("Count: " + i);
}

// While loop
int i = 1;
while (i <= 5) {
    System.out.println(i);
    i++;
}

// For-each — cleanest way to iterate arrays
int[] scores = { 85, 90, 78, 92 };
for (int score : scores) {
    System.out.println(score);
}

// FizzBuzz — classic exercise
for (int n = 1; n <= 20; n++) {
    if      (n % 15 == 0) System.out.println("FizzBuzz");
    else if (n % 3  == 0) System.out.println("Fizz");
    else if (n % 5  == 0) System.out.println("Buzz");
    else                 System.out.println(n);
}

Clean Coding

What is Clean Code?

Clean code is code that is easy to read, understand, and maintain. It reads almost like prose. The goal is to write code for humans first, computers second.

❌ Dirty Code

int x = 5;
int y = 12;
double z;
z = x * y * 0.0833;
System.out.println(z);

✅ Clean Code

final double HOURLY_RATE = 0.0833;
int hoursWorked = 5;
int daysInMonth = 12;
double monthlySalary =
  hoursWorked * daysInMonth * HOURLY_RATE;
System.out.println(monthlySalary);

Creating Methods

A method is a reusable block of code that performs one specific task. Methods reduce repetition (DRY principle) and improve readability.

Java — Method Anatomy
public class Calculator {

    // Method with return value
    static double add(double a, double b) {
        return a + b;
    }

    // Method with no return (void)
    static void printGreeting(String name) {
        System.out.println("Hello, " + name + "!");
    }

    // Overloading — same name, different parameters
    static int multiply(int a, int b) { return a * b; }
    static double multiply(double a, double b) { return a * b; }

    public static void main(String[] args) {
        System.out.println(add(3.5, 2.5));       // 6.0
        printGreeting("Alice");                // Hello, Alice!
        System.out.println(multiply(3, 4));    // 12 (int version)
    }
}
Method Design PrinciplesA method should do one thing only. If you can't describe what it does without using "and", it should be split. Keep methods under ~20 lines. Name them with a verb: calculateTotal, printReport, isValidAge.

Refactoring — Before & After

Java — After Refactoring ✅
public class MortgageCalculator {
    static final int MONTHS_IN_YEAR = 12;
    static final int PERCENT = 100;

    static double calculateMortgage(int principal, float annualRate, byte years) {
        double monthlyRate = getMonthlyRate(annualRate);
        int numPayments = getNumberOfPayments(years);
        return principal
            * (monthlyRate * Math.pow(1 + monthlyRate, numPayments))
            / (Math.pow(1 + monthlyRate, numPayments) - 1);
    }

    private static double getMonthlyRate(float annualRate) {
        return annualRate / PERCENT / MONTHS_IN_YEAR;
    }

    private static int getNumberOfPayments(byte years) {
        return years * MONTHS_IN_YEAR;
    }

    public static void main(String[] args) {
        double mortgage = calculateMortgage(300_000, 3.92f, (byte)30);
        System.out.printf("Monthly Payment: $%.2f%n", mortgage);
    }
}
Output
Monthly Payment: $1,415.29

Debugging & Deploying

Types of Errors

TypeWhen DetectedExampleFix
Syntax ErrorAt compile timeMissing ; or }IDE underlines it — just fix it
Runtime ErrorWhile runningNullPointerException, divide by 0Add null checks, validate inputs
Logic ErrorWrong outputUsing < instead of <=Use the debugger, add print statements

Common Mistakes

Java — Common Errors
// ❌ Missing semicolon
int x = 5   // compile error: ';' expected

// ❌ Array out of bounds (runtime error)
int[] arr = { 1, 2, 3 };
System.out.println(arr[5]);  // ArrayIndexOutOfBoundsException

// ❌ NullPointerException
String s = null;
System.out.println(s.length()); // NPE — can't call methods on null

// ✅ Fix: null check before use
if (s != null)
    System.out.println(s.length());

Debugging in IntelliJ IDEA

The debugger lets you pause execution at any line (breakpoint) and inspect the values of all variables. This is far more powerful than adding System.out.println() everywhere.

ActionMacWindowsWhat it does
Set Breakpoint⌘F8Ctrl+F8Marks a line to pause at
Start Debugger⌃DShift+F9Runs program in debug mode
Step OverF8F8Execute current line, go to next
Step IntoF7F7Enter the method being called
Resume⌘⌥RF9Continue until next breakpoint
Debugging StrategySet a breakpoint just before the bug. Run in debug mode. Use Step Over to walk through line by line, watching the Variables panel. When a variable has an unexpected value, you've found the bug.

Packaging Java Applications (JAR)

A JAR (Java ARchive) bundles all your compiled .class files into a single distributable file. Anyone with a JRE can run it with java -jar.

Terminal — Build & Run JAR
# Compile all Java files
$ javac -d out src/*.java

# Create JAR with a manifest specifying Main class
$ jar cfe MyApp.jar com.example.Main -C out .

# Run the JAR
$ java -jar MyApp.jar
