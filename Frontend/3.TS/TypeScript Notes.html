Chapter 1 · TypeScript Core

Getting Started

What is TypeScript? Intro

TypeScript is a typed superset of JavaScript developed by Microsoft. It compiles down to plain JavaScript, adding static type checking, interfaces, generics, and enhanced tooling. Every valid JavaScript file is also valid TypeScript.

JavaScript

// No type info — bug at runtime
function add(a, b) {
  return a + b;
}
add("5", 3); // "53" — wrong!

TypeScript

// Types catch bug at compile time
function add(a: number, b: number): number {
  return a + b;
}
add("5", 3); // ❌ Error before running!
Key Benefits TypeScript catches errors at compile time (not runtime), provides intelligent IDE autocomplete, makes large codebases maintainable, and documents code via types — all while outputting standard JS.
Installation & Setup

Install TypeScript globally via npm. The tsc command compiles .ts files to .js.

Terminal — Setup
# Install TypeScript globally
npm install -g typescript

# Verify installation
tsc --version       # Version 5.x.x

# Compile a single file
tsc hello.ts        # → produces hello.js

# Watch mode (re-compiles on save)
tsc hello.ts --watch

# Initialize a project config
tsc --init          # → creates tsconfig.json
tsconfig.json — The TypeScript Compiler Config

The tsconfig.json file controls how TypeScript compiles your project — output folder, JS version target, strictness settings, and more.

tsconfig.json
{
  "compilerOptions": {
    "target": "ES2016",       // Output JS version
    "module": "commonjs",    // Module system
    "rootDir": "./src",      // Input folder
    "outDir": "./dist",      // Output folder
    "strict": true,          // Enable all strict checks
    "noImplicitAny": true,   // Disallow implicit 'any'
    "sourceMap": true,       // For debugging
    "removeComments": true  // Strip comments in output
  }
}
OptionPurposeRecommended
strictEnables all strict type checks at oncetrue
noImplicitAnyError when type is inferred as anytrue
strictNullChecksPrevents assigning null/undefined to typed varstrue
targetWhich JS version to compile toES2016+
sourceMapGenerates .map files for debugging TS in browsertrue
Debugging TypeScript

Set "sourceMap": true in tsconfig, then configure your debugger (VS Code) to use the .ts sources instead of compiled .js.

VS Code Debugging In launch.json, set "preLaunchTask": "tsc: build" and "outFiles": ["${workspaceFolder}/dist/**/*.js"]. You can then set breakpoints directly in .ts files.

Chapter 2 · TypeScript Core

Fundamentals

Built-in Types Core

TypeScript extends JavaScript's primitive types with explicit type annotations. The compiler uses these to check correctness before your code runs.

TypeScript — Primitive Types
// number — all numerics (int, float, hex, binary, octal)
let age: number = 25;
let price: number = 9.99;
let hex: number = 0xff;

// string
let firstName: string = 'Alice';
let greeting: string = `Hello, ${firstName}!`;

// boolean
let isActive: boolean = true;
let isAdmin: boolean = false;

// Type annotations are often optional — TypeScript can INFER types
let score = 100;   // inferred as number
let name = 'Bob'; // inferred as string

// Explicit annotation useful when initializing later
let level: number;
level = 5; // OK
level = 'five'; // ❌ Error: Type 'string' is not assignable to 'number'
The any Type Avoid

any turns off TypeScript's type checking for a variable — it can hold anything. It's a last resort for gradual migration or unknown data. Overusing any defeats the purpose of TypeScript.

TypeScript — any type
let data: any = 42;
data = 'hello'; // OK — any accepts all
data = true;
data.someMethod(); // No error — but may crash at runtime!

// Implicit any — happens when noImplicitAny is off
function process(value) { // 'value' is implicitly 'any'
  value.doSomething(); // No error, but risky
}
Avoid any Using any disables all type safety. Use unknown instead when the type is genuinely unknown — it forces you to narrow the type before using it.
Arrays & Tuples

TypeScript arrays are typed — all elements must match the declared type. Tuples are fixed-length arrays where each position has a specific type.

TypeScript — Arrays
// Two equivalent array syntaxes
let numbers: number[] = [1, 2, 3];
let strings: Array<string> = ['a', 'b'];

// Inferred array type
let scores = [90, 85, 92]; // inferred: number[]

// Accessing elements is typed
let first = numbers[0]; // type: number

// ❌ Type error
numbers.push('hello'); // Error: Argument of type 'string'
TypeScript — Tuples
// Tuple: fixed-length, each position typed
let user: [number, string] = [1, 'Alice'];

// Destructure tuple
const [id, username] = user;
console.log(id);       // 1  (type: number)
console.log(username); // 'Alice' (type: string)

// Common use: key-value pair
let entry: [string, number] = ['score', 100];

// ❌ Wrong order gives error
let bad: [string, number] = [100, 'score']; // Error!
Enums Named Constants

Enums define a set of named constants. They make code more readable by replacing magic numbers or strings with meaningful names.

TypeScript — Enums
// Numeric enum (auto-increments from 0)
enum Direction {
  Up,    // 0
  Down,  // 1
  Left,  // 2
  Right  // 3
}

let move: Direction = Direction.Up;
console.log(move); // 0

// String enum (preferred — more readable in debug)
enum Size {
  Small  = 'S',
  Medium = 'M',
  Large  = 'L'
}

function getShirt(size: Size) {
  return `Shirt size: ${size}`;
}
getShirt(Size.Medium); // 'Shirt size: M'

// const enum — inlined at compile time (no runtime object)
const enum Status { Active = 1, Inactive = 0 }
Functions Typed Params & Return

TypeScript allows you to annotate function parameter types and return types, catching wrong arguments and return values at compile time.

TypeScript — Functions
// Basic typed function
function greet(name: string): string {
  return `Hello, ${name}!`;
}

// Optional parameter — use ?
function buildURL(host: string, port?: number): string {
  return port ? `${host}:${port}` : host;
}
buildURL('localhost');       // OK
buildURL('localhost', 3000); // OK

// Default parameter
function tax(price: number, rate: number = 0.1): number {
  return price * (1 + rate);
}

// void return — function returns nothing
function log(msg: string): void {
  console.log(msg);
}

// Arrow function with types
const multiply = (x: number, y: number): number => x * y;

// Rest parameters
function sum(...nums: number[]): number {
  return nums.reduce((a, b) => a + b, 0);
}
sum(1, 2, 3, 4); // 10
Objects

Object type annotations define the shape of an object — which properties it must have and their types.

TypeScript — Object Types
// Inline object type annotation
let employee: { id: number; name: string; retired?: boolean };
employee = { id: 1, name: 'Alice' }; // OK — retired is optional

// Function that takes an object
function printEmployee(emp: { id: number; name: string }): void {
  console.log(`[${emp.id}] ${emp.name}`);
}

// readonly property — cannot be changed after creation
let config: { readonly apiUrl: string } = { apiUrl: 'https://api.example.com' };
config.apiUrl = 'other'; // ❌ Error: readonly

Chapter 3 · TypeScript Core

Advanced Types

Type Aliases Reusable Types

A type alias gives a name to any type — primitives, objects, unions, tuples, functions. It improves readability and avoids repeating complex type shapes.

TypeScript — Type Aliases
// Alias for a primitive
type ID = number;
let userId: ID = 101;

// Alias for an object shape
type Point = { x: number; y: number };
let origin: Point = { x: 0, y: 0 };

// Alias for a function signature
type Formatter = (value: number) => string;
const toUSD: Formatter = (v) => `$${v.toFixed(2)}`;

// Alias for a complex nested shape
type User = {
  id: number;
  name: string;
  address: {
    city: string;
    country: string;
  };
};
const user: User = { id: 1, name: 'Alice', address: { city: 'Paris', country: 'FR' } };
Union Types

A union type (A | B) allows a variable to hold one of several types. Use type narrowing (typeof, instanceof, in) to work with the specific type inside a branch.

TypeScript — Union Types
// Union type
type StringOrNumber = string | number;
let input: StringOrNumber = 'hello';
input = 42; // Also OK

// Type narrowing with typeof
function format(value: string | number): string {
  if (typeof value === 'string') {
    return value.toUpperCase(); // value is string here
  }
  return value.toFixed(2); // value is number here
}

// Discriminated union — great for state modelling
type Shape =
  | { kind: 'circle'; radius: number }
  | { kind: 'rectangle'; width: number; height: number };

function area(shape: Shape): number {
  switch (shape.kind) {
    case 'circle':    return Math.PI * shape.radius ** 2;
    case 'rectangle': return shape.width * shape.height;
  }
}
Intersection Types

An intersection type (A & B) combines multiple types into one. The result must satisfy all combined types simultaneously — useful for mixins and composition.

TypeScript — Intersection Types
type Draggable = { drag(): void };
type Resizable = { resize(): void };

// Intersection: must implement BOTH drag and resize
type Widget = Draggable & Resizable;

const textBox: Widget = {
  drag() { console.log('dragging'); },
  resize() { console.log('resizing'); }
};
Literal Types

Literal types restrict a variable to an exact value — not just any string or number. Combined with unions, they create precise, self-documenting APIs.

TypeScript — Literal Types
// Exact string literals only
type Direction = 'left' | 'right' | 'up' | 'down';

function move(dir: Direction) {
  console.log(`Moving ${dir}`);
}
move('left');     // ✅
move('diagonal'); // ❌ Error

// Numeric literals
type DiceRoll = 1 | 2 | 3 | 4 | 5 | 6;
let roll: DiceRoll = 4;
roll = 7; // ❌ Error

// Mixed literal + type
type Margin = number | 'auto';
Nullable Types, Optional Chaining & Nullish Coalescing

With strictNullChecks enabled, null and undefined are separate types. Use optional chaining (?.) to safely access nested properties, and nullish coalescing (??) as a null-safe fallback.

TypeScript — Null Safety
// Nullable type — must include null explicitly
let name: string | null = null;
name = 'Alice'; // OK

// Optional chaining — stops if null/undefined
type Customer = { birthday?: { year: number } };
const c: Customer = {};

// Without optional chaining — throws if birthday is undefined
// const year = c.birthday.year; ← ❌ Runtime error

// With optional chaining — returns undefined safely
const year = c.birthday?.year; // undefined

// Nullish coalescing — fallback ONLY for null/undefined
const displayYear = year ?? 2000; // 2000

// ?? vs || difference
console.log(0 || 'default');  // 'default' — 0 is falsy
console.log(0 ?? 'default');  // 0 — only null/undefined trigger ??
Type Assertions, unknown & never

Type assertions let you tell the compiler "I know the type better". unknown is the safe version of any. never represents values that never occur — useful for exhaustive checks.

TypeScript — Assertions & Special Types
// Type assertion — tells TS what type to treat as
let input = document.getElementById('name') as HTMLInputElement;
input.value = 'Alice'; // OK — TS knows it's an input

// unknown — safer than any
function processInput(val: unknown) {
  // Must narrow before using
  if (typeof val === 'string') {
    console.log(val.toUpperCase()); // OK inside narrowed block
  }
  val.doSomething(); // ❌ Error — can't call without narrowing
}

// never — exhaustiveness checking
type Status = 'active' | 'inactive';

function handle(s: Status) {
  switch (s) {
    case 'active':   return 'active!';
    case 'inactive': return 'inactive!';
    default:
      const _check: never = s; // Error if a case is missed
  }
}

Chapter 4 · Object-Oriented TypeScript

Classes & OOP

Creating Classes & Objects OOP

TypeScript classes add type annotations to ES6 classes. Properties must be declared with types before use. The constructor initializes them.

TypeScript — Class Basics
class Account {
  // Property declarations (required in TS)
  id: number;
  owner: string;
  balance: number;

  constructor(id: number, owner: string, balance: number) {
    this.id = id;
    this.owner = owner;
    this.balance = balance;
  }

  deposit(amount: number): void {
    if (amount <= 0) throw new Error('Invalid amount');
    this.balance += amount;
  }

  toString(): string {
    return `Account[${this.id}] ${this.owner}: $${this.balance}`;
  }
}

const acc = new Account(1, 'Alice', 1000);
acc.deposit(500);
console.log(acc.toString()); // Account[1] Alice: $1500
Access Modifiers, readonly & Parameter Properties

public, private, and protected control visibility. Parameter properties are a TypeScript shortcut that declares and assigns properties directly in the constructor signature.

TypeScript — Modifiers & Parameter Properties
class Account {
  // Parameter properties — TS shortcut!
  // Declares AND assigns all at once
  constructor(
    public readonly id: number,
    public owner: string,
    private _balance: number
  ) {}

  // Getter — access private field
  get balance(): number { return this._balance; }

  // Setter — with validation
  set balance(value: number) {
    if (value < 0) throw new Error('Negative balance');
    this._balance = value;
  }
}

const a = new Account(1, 'Alice', 500);
console.log(a.balance); // 500
a.balance = 750;        // calls setter
a.id = 2;               // ❌ Error: readonly

protectedThis class + subclassesShared state for inheritance
readonlyCan read anywhere, can't reassignImmutable IDs, config
Static Members & Index Signatures
TypeScript — Static & Index Signatures
class MathUtils {
  static PI = 3.14159;

  static circleArea(r: number): number {
    return MathUtils.PI * r ** 2;
  }
}
MathUtils.circleArea(5); // 78.54 — called on class, not instance

// Index signatures — dynamic property keys
class SeatAssignment {
  [seatNumber: string]: string; // any string key → string value
}

const seats = new SeatAssignment();
seats['A1'] = 'Alice';
seats['B2'] = 'Bob';
Inheritance, Method Overriding & Polymorphism

Classes can extend other classes with extends. Override methods with the override keyword (TS4+). Polymorphism lets you treat subclass instances uniformly through the base type.

TypeScript — Inheritance & Polymorphism
class Animal {
  constructor(public name: string) {}

  makeSound(): void {
    console.log('...');
  }
}

class Dog extends Animal {
  constructor(name: string, public breed: string) {
    super(name); // must call super() first
  }

  override makeSound(): void {
    console.log('Woof!');
  }
}

class Cat extends Animal {
  override makeSound(): void { console.log('Meow!'); }
}

// Polymorphism — same method, different behavior
const animals: Animal[] = [new Dog('Rex', 'Lab'), new Cat('Mia')];
animals.forEach(a => a.makeSound()); // Woof! Meow!
Abstract Classes & Interfaces

Abstract classes define a common blueprint but can't be instantiated. Interfaces describe the shape of objects/classes — purely a compile-time contract with no runtime overhead.

TypeScript — Abstract Classes
abstract class Shape {
  constructor(public color: string) {}

  // Abstract method — subclasses MUST implement
  abstract area(): number;

  // Concrete method — shared by all subclasses
  describe(): string {
    return `A ${this.color} shape with area ${this.area()}`;
  }
}

class Circle extends Shape {
  constructor(color: string, public radius: number) {
    super(color);
  }
  override area() { return Math.PI * this.radius ** 2; }
}

const c = new Circle('red', 5);
console.log(c.describe()); // A red shape with area 78.54
TypeScript — Interfaces
// Interface — a contract for shape
interface Calendar {
  name: string;
  addEvent(event: string): void;
  removeEvent(event: string): void;
}

class GoogleCalendar implements Calendar {
  constructor(public name: string) {}
  addEvent(e: string) { console.log(`Adding ${e} to Google`); }
  removeEvent(e: string) { console.log(`Removing ${e}`); }
}

// Interface vs Type: interfaces can be merged, types can't
interface User { name: string; }
interface User { age: number; }  // merges with first User
// User now has both name and age

Chapter 5 · Object-Oriented TypeScript

Generics

The Problem Generics Solve Reusability

Without generics, you'd either duplicate code for each type, or use any and lose type safety. Generics let you write code that works with any type while keeping full type safety.

TypeScript — Without vs With Generics
// Without generics — forced to duplicate
function wrapNumber(val: number): { value: number } { return { value: val }; }
function wrapString(val: string): { value: string } { return { value: val }; }

// With generics — one function handles all types!
function wrap<T>(val: T): { value: T } {
  return { value: val };
}

const n = wrap(42);        // T inferred as number → { value: number }
const s = wrap('hello');  // T inferred as string → { value: string }
s.value.toUpperCase();   // ✅ TypeScript knows it's a string!
Generic Classes & Interfaces
TypeScript — Generic Class
// Generic class — Stack data structure
class Stack<T> {
  private items: T[] = [];

  push(item: T): void { this.items.push(item); }

  pop(): T | undefined { return this.items.pop(); }

  peek(): T | undefined { return this.items[this.items.length - 1]; }

  get size(): number { return this.items.length; }

  isEmpty(): boolean { return this.items.length === 0; }
}

const numStack = new Stack<number>();
numStack.push(1);
numStack.push(2);
console.log(numStack.pop()); // 2 — TypeScript knows it's number

// Generic interface
interface Repository<T> {
  findById(id: number): T | undefined;
  save(entity: T): void;
  delete(id: number): void;
}
Generic Constraints & keyof

Constraints (T extends Something) restrict which types can be used as the generic argument. keyof produces the union of keys of a type.

TypeScript — Constraints & keyof
// Constrain T to have a .length property
function longest<T extends { length: number }>(a: T, b: T): T {
  return a.length >= b.length ? a : b;
}
longest([1, 2, 3], [1, 2]);       // [1,2,3]
longest('hello', 'hi');          // 'hello'

// keyof — keys of an object type as a union
function pluck<T, K extends keyof T>(obj: T, key: K): T[K] {
  return obj[key];
}

const person = { name: 'Alice', age: 30 };
pluck(person, 'name'); // 'Alice' — typed as string
pluck(person, 'age');  // 30 — typed as number
pluck(person, 'x');   // ❌ Error: 'x' not a key of person
Type Mapping

Mapped types transform every property of a type. TypeScript's built-in utilities like Partial, Required, Readonly, and Pick are all built using mapped types.

TypeScript — Mapped Types
// Custom mapped type — make all properties optional
type Optional<T> = { [K in keyof T]?: T[K] };

// Built-in utility types
type User = { name: string; email: string; age: number };

type PartialUser  = Partial<User>;    // all optional
type ReadonlyUser = Readonly<User>;   // all readonly
type NameOnly    = Pick<User, 'name' | 'email'>; // subset
type WithoutAge  = Omit<User, 'age'>;  // exclude field

// Real world — update function only needs changed fields
function updateUser(id: number, fields: Partial<User>): void {
  console.log(`Updating user ${id}`, fields);
}
updateUser(1, { name: 'Bob' }); // Only update name — OK!

Chapter 6 · Object-Oriented TypeScript

Decorators

What Are Decorators? Experimental

Decorators are special functions prefixed with @ that augment classes, methods, properties, or parameters. They are widely used in frameworks like Angular and NestJS.

Enable in tsconfig Set "experimentalDecorators": true in tsconfig.json to use decorators.
TypeScript — Class Decorator
// A class decorator is a function that receives the constructor
function Component(constructor: Function) {
  constructor.prototype.uniqueId = Date.now();
}

@Component
class ProfileComponent {
  name: string = 'Profile';
}

// Parameterized decorator — returns a decorator
function ComponentWithOptions(options: { selector: string }) {
  return (constructor: Function) => {
    constructor.prototype.selector = options.selector;
  };
}

@ComponentWithOptions({ selector: 'app-profile' })
class AppProfile {}
Method, Property & Parameter Decorators
TypeScript — Method Decorator
// Method decorator — wraps method with extra behavior
function Log(target: any, key: string, desc: PropertyDescriptor) {
  const original = desc.value;
  desc.value = function(...args: any[]) {
    console.log(`Calling ${key} with`, args);
    const result = original.apply(this, args);
    console.log(`${key} returned`, result);
    return result;
  };
}

class Calculator {
  @Log
  add(a: number, b: number): number { return a + b; }
}
new Calculator().add(2, 3);
// Calling add with [2, 3]
// add returned 5
TypeScript — Property Decorator
// Property decorator — metadata or validation
function MinLength(min: number) {
  return (target: any, key: string) => {
    let value: string;
    Object.defineProperty(target, key, {
      get: () => value,
      set: (v: string) => {
        if (v.length < min) throw new Error(`Min length is ${min}`);
        value = v;
      }
    });
  };
}

class User {
  @MinLength(3)
  username: string = '';
}

Modules

Exporting & Importing ES Modules

Modules let you split code into reusable files. TypeScript uses the same ES Module syntax as modern JavaScript. Each file is its own scope by default.

shapes.ts — Exporting
// Named exports
export type Point = { x: number; y: number };

export function distance(a: Point, b: Point): number {
  return Math.sqrt((b.x - a.x) ** 2 + (b.y - a.y) ** 2);
}

// Default export — one per file
export default class Circle {
  constructor(public radius: number) {}
  area(): number { return Math.PI * this.radius ** 2; }
}
app.ts — Importing
// Import named exports
import { Point, distance } from './shapes';

// Import with alias
import { distance as dist } from './shapes';

// Import default
import Circle from './shapes';

// Import both
import Circle, { distance } from './shapes';

// Import all as namespace
import * as Shapes from './shapes';
const c = new Shapes.default(5);
Re-exporting & Barrel Files

Re-exporting lets you create "barrel" index.ts files that group and re-export from many modules, giving consumers a single clean import path.

index.ts — Barrel / Re-export
// Re-export everything from sub-modules
export { Circle, distance } from './shapes';
export { Stack, Queue }    from './dataStructures';
export *                     from './utils';

// Consumers now import from one place
import { Circle, Stack, distance } from './index';

JavaScript Integration

Including JS in TS Projects & Type Checking JS Migration

TypeScript can gradually type-check JavaScript files. Enable allowJs to include .js files in a TS project, and checkJs to report errors in them too.

tsconfig.json — JS Integration
{
  "compilerOptions": {
    "allowJs": true,    // Include .js files in compilation
    "checkJs": true,    // Type-check .js files
    "declaration": true // Generate .d.ts files
  }
}
JSDoc Type Annotations in JS files

Use JSDoc comments to annotate JavaScript files with types — TypeScript will read them without requiring you to rename files to .ts.

JavaScript with JSDoc Types
/**
 * @param {number} a
 * @param {number} b
 * @returns {number}
 */
function add(a, b) {
  return a + b;
}

/**
 * @typedef {{ name: string; age: number }} Person
 */

/**
 * @param {Person} person
 * @returns {string}
 */
function greet(person) {
  return `Hello, ${person.name}!`;
}
Declaration Files (.d.ts) & DefinitelyTyped

Declaration files (.d.ts) describe the types of existing JavaScript libraries without modifying them. The community maintains declarations for thousands of packages at DefinitelyTyped.

Typing Components & Props

Define a type or interface for component props. TypeScript will enforce correct usage wherever the component is used.

Button.tsx — Typed Component
import { FC } from 'react';

interface ButtonProps {
  label: string;
  onClick: () => void;
  variant?: 'primary' | 'secondary' | 'danger';
  disabled?: boolean;
}

const Button: FC<ButtonProps> = ({
  label, onClick, variant = 'primary', disabled = false
}) => {
  return (
    <button
      className={`btn btn-${variant}`}
      onClick={onClick}
      disabled={disabled}
    >
      {label}
    </button>
  );
};

export default Button;

useState & useEffect with Types
TypeScript — Typed Hooks
import { useState, useEffect } from 'react';

interface User { id: number; name: string; email: string }

function UserList() {
  // State with generic type — TypeScript infers User[]
  const [users, setUsers] = useState<User[]>([]);
  const [loading, setLoading] = useState<boolean>(true);
  const [error, setError] = useState<string | null>(null);

  useEffect(() => {
    fetch('/api/users')
      .then(r => r.json())
      .then((data: User[]) => { setUsers(data); setLoading(false); })
      .catch((e: Error) => setError(e.message));
  }, []);

  if (loading) return <p>Loading...</p>;
  if (error)   return <p>Error: {error}</p>;

  return (
    <ul>
      {users.map(u => <li key={u.id}>{u.name}</li>)}
    </ul>
  );
}
Handling Events & Forms
TypeScript — Typed Events & Forms
import { useState } from 'react';

function LoginForm() {
  const [email, setEmail] = useState('');
  const [password, setPassword] = useState('');

  // React.ChangeEvent typed to the element type
  const handleChange = (e: React.ChangeEvent<HTMLInputElement>) => {
    const { name, value } = e.target;
    if (name === 'email') setEmail(value);
    if (name === 'password') setPassword(value);
  };

  const handleSubmit = (e: React.FormEvent<HTMLFormElement>) => {
    e.preventDefault();
    console.log('Login', { email, password });
  };

  return (
    <form onSubmit={handleSubmit}>
      <input name="email"    value={email}    onChange={handleChange} />
      <input name="password" value={password} onChange={handleChange} />
      <button type="submit">Login</button>
    </form>
  );
}

Building a Typed REST API

Use Router to split routes into files. Type request params, query, and body for fully safe route handlers.

routes/users.ts — Typed Router
import { Router, Request, Response } from 'express';

interface User { id: number; name: string; email: string }

const router = Router();
let users: User[] = [{ id: 1, name: 'Alice', email: 'alice@example.com' }];

// GET /users — typed response
router.get('/', (req: Request, res: Response<User[]>) => {
  res.json(users);
});

// GET /users/:id — typed params
router.get('/:id', (req: Request<{ id: string }>, res: Response) => {
  const user = users.find(u => u.id === parseInt(req.params.id));
  if (!user) return res.status(404).json({ error: 'Not found' });
  res.json(user);
});

// POST /users — typed request body
router.post('/', (req: Request<{}, {}, Omit<User, 'id'>>, res: Response) => {
  const newUser: User = { id: users.length + 1, ...req.body };
  users.push(newUser);
  res.status(201).json(newUser);
});

export default router;
