Getting Started

Chapter 1 · JavaScript Basics
What is JavaScript? Intro

JavaScript is a lightweight, interpreted, object-oriented programming language primarily used to add interactivity to web pages. It runs in the browser and also server-side via Node.js. JS is dynamically typed — you don't declare types; they are determined at runtime.

Key Facts JavaScript was created in 10 days by Brendan Eich in 1995. Despite sharing a name, it has nothing to do with Java. It is now standardized as ECMAScript (ES). Modern JS = ES6+ features.
JS in the Browser vs Node.js Environment

JavaScript can run in two environments. The browser provides DOM APIs for interacting with HTML. Node.js provides file system, network, and server APIs.

Browser JS

// Access the DOM
const btn = document.getElementById('myBtn');
btn.addEventListener('click', () => {
  document.body.style.background = 'blue';
});

Node.js

// Access the file system
const fs = require('fs');
fs.readFile('hello.txt', 'utf8',
  (err, data) => console.log(data)
);
Separation of Concerns

Keep HTML (structure), CSS (style), and JavaScript (behavior) in separate files. Never mix JavaScript inline in HTML — it makes code harder to maintain and reuse.

Best Practice Always place <script> tags at the bottom of <body> so the HTML loads before JS executes.

Variables & Types

Chapter 2 · JavaScript Basics
var, let, const Core

JavaScript has three ways to declare variables. Prefer const by default; use let when the value needs to change; avoid var (function-scoped, hoisted in confusing ways).

JavaScript 
// var — function-scoped, avoid in modern JS
var score = 10;
score = 20;      // can reassign
var score = 30;  // can even re-declare!

// let — block-scoped, reassignable
let count = 0;
count = 1;        // OK
// let count = 5; ← ERROR: already declared

// const — block-scoped, NOT reassignable
const PI = 3.14159;
// PI = 3; ← TypeError: Assignment to constant

// BUT: const objects/arrays can be mutated!
const person = { name: 'Alice' };
person.name = 'Bob';  // ✅ OK — mutating the object
// person = {}; ← ❌ ERROR — reassigning the variable

letBlockYesNo (TDZ)Mutable values
constBlockNoNo (TDZ)Default — most of the time
Primitive Types Types

JavaScript has 7 primitive types. Primitives are immutable — they cannot be altered after creation. They are stored by value (copied when assigned).

JavaScript — All Primitive Types
// 1. String
let name    = 'Alice';          // single quotes
let greet   = "Hello";          // double quotes
let msg     = `Hi ${name}!`;   // template literal

// 2. Number (integers AND floats — same type)
let age     = 25;
let price   = 9.99;
let inf     = Infinity;
let nan     = NaN;             // "Not a Number"

// 3. Boolean
let isOnline = true;
let hasError = false;

// 4. undefined — declared but not assigned
let x;
console.log(x);               // undefined

// 5. null — intentional absence of a value
let user = null;

// 6. Symbol — unique identifier (ES6)
const id = Symbol('id');

// 7. BigInt — very large integers (ES2020)
const big = 9007199254740991n;

Dynamic Typing

JavaScript automatically determines and can change a variable's type at runtime. This is powerful but can lead to bugs — always be aware of what type a variable holds.

JavaScript
let x = 42;
console.log(typeof x);    // "number"

x = "hello";
console.log(typeof x);    // "string"

x = true;
console.log(typeof x);    // "boolean"

// Implicit type coercion — JavaScript converts types automatically
console.log("5" + 3);      // "53" — number becomes string
console.log("5" - 3);      // 2   — string becomes number
console.log(true + 1);    // 2   — true becomes 1
Objects Reference Type

Objects group related data and functions together. They are stored by reference — assigning an object to a variable stores its memory address, not a copy.

JavaScript
// Object literal syntax
const person = {
  name: 'Alice',
  age: 30,
  isAdmin: false,
  address: {                   // nested object
    city: 'Paris',
    country: 'France'
  },
  greet() {                    // method
    return `Hi, I'm ${this.name}`;
  }
};

// Dot notation
console.log(person.name);          // 'Alice'
console.log(person.address.city);  // 'Paris'

// Bracket notation (dynamic keys)
let key = 'age';
console.log(person[key]);            // 30

// Reference behavior
const p2 = person;  // p2 points to the SAME object
p2.name = 'Bob';
console.log(person.name);  // 'Bob' — person changed too!

Arrays are ordered collections that can hold any type of value. They are objects under the hood — stored by reference.

JavaScript

// Accessing elements (0-indexed)

const mixed = [1, 'hello', true, null, { x: 5 }];

Equality Operators Important!

JavaScript has two equality operators. Always prefer === (strict equality) which checks both value AND type.

== (Loose / Abstract)

// Converts types before comparing
1 == '1'      // true  ← dangerous
0 == false    // true
null == undefined // true
'' == false   // true

=== (Strict) ✅ Use This

// No type conversion
1 === '1'     // false — different types
1 === 1      // true
null === undefined // false
false === false // true
Ternary Operator

A compact way to write if-else as an expression: condition ? valueIfTrue : valueIfFalse

JavaScript
const age = 20;

// Traditional if-else
let status;
if (age >= 18) status = 'adult';
else status = 'minor';

// Same with ternary — clean and concise
const status2 = age >= 18 ? 'adult' : 'minor';

// Nested ternary (use sparingly — gets confusing)
const grade = 75;
const letter = grade >= 90 ? 'A'
             : grade >= 80 ? 'B'
             : grade >= 70 ? 'C' : 'F';
// 'C'
Logical Operators
JavaScript
// && (AND) — true if BOTH sides are truthy
console.log(true && true);    // true
console.log(true && false);   // false

// || (OR) — true if EITHER side is truthy
console.log(false || true);   // true
console.log(false || false);  // false

// ! (NOT) — flips the boolean
console.log(!true);           // false
console.log(!false);          // true

// With non-booleans — SHORT CIRCUIT EVALUATION
// && returns first falsy OR last value
console.log(1 && 2 && 3);     // 3
console.log(1 && null && 3);  // null

// || returns first truthy OR last value
console.log(null || 'default');  // 'default'
console.log('Bob' || 'default');  // 'Bob'

// Nullish coalescing ?? — only falls back on null/undefined
console.log(0 ?? 'default');  // 0  (0 is not null/undefined)
console.log(null ?? 'default'); // 'default'
Practical Use of Short-Circuit const displayName = user.name || 'Guest'; — provides a fallback when name is falsy. Use ?? when 0 or empty string are valid values.
Operator Precedence

Operators have a priority order (like PEMDAS in math). Parentheses always win.

JavaScript
// Without parentheses — follows precedence rules
console.log(2 + 3 * 4);     // 14 (* before +)
console.log(10 - 2 ** 3);   // 2  (** before -)

// With parentheses — explicit order
console.log((2 + 3) * 4);   // 20

// Precedence: ** > * / % > + - > comparisons > === > && > || > ?:

  console.log('No name');   // ← empty string is falsy

Switch Statement

Use switch when comparing one value against multiple exact options. Always include break to prevent fall-through.

JavaScript
const day = 'Monday';

switch (day) {
  case 'Monday':
  case 'Tuesday':
  case 'Wednesday':
  case 'Thursday':
  case 'Friday':
    console.log('Weekday');   // fall-through for grouping
    break;
  case 'Saturday':
  case 'Sunday':
    console.log('Weekend!');
    break;
  default:
    console.log('Unknown day');
}
Forgetting break Without break, execution falls through to the next case. This is almost always a bug — always include break (or return inside a function).
Loops

JavaScript has 5 main loop constructs. Choose based on what you're iterating over.

for loop — classic counting loop
// for (init; condition; update)
for (let i = 0; i < 5; i++) {
  console.log(i);   // 0, 1, 2, 3, 4
}

// Reverse loop
for (let i = 4; i >= 0; i--) {
  console.log(i);   // 4, 3, 2, 1, 0
}
while & do...while
// while — checks condition BEFORE each iteration
let i = 0;
while (i < 3) {
  console.log(i++);  // 0, 1, 2
}

// do...while — executes AT LEAST ONCE
let j = 10;
do {
  console.log('runs once');  // even though j > 3
} while (j < 3);
for...in and for...of
// for...in — iterates over KEYS of an object
const person = { name: 'Alice', age: 30 };
for (const key in person) {
  console.log(key, person[key]);
  // 'name' 'Alice'
  // 'age' 30
}

// for...of — iterates over VALUES of an iterable
const colors = ['red', 'green', 'blue'];
for (const color of colors) {
  console.log(color);  // 'red', 'green', 'blue'
}

// for...of with strings too!
for (const char of 'hello') {
  console.log(char);  // 'h', 'e', 'l', 'l', 'o'
}
break & continue
// break — exits the loop entirely
for (let i = 0; i < 10; i++) {
  if (i === 5) break;
  console.log(i);  // 0, 1, 2, 3, 4
}

// continue — skips current iteration
for (let i = 0; i < 5; i++) {
  if (i % 2 === 0) continue;
  console.log(i);  // 1, 3 — skips even numbers
}

Prime Number Check 
function isPrime(n) {
  if (n < 2) return false;
  for (let i = 2; i <= Math.sqrt(n); i++) {
    if (n % i === 0) return false;  // divisible — not prime
  }
  return true;
}

console.log(isPrime(7));   // true
console.log(isPrime(10));  // false
console.log(isPrime(2));   // true

Objects Deep Dive

Chapter 5 · JavaScript Basics
Factory Functions

A factory function is a regular function that creates and returns a new object. Use when you need to create multiple similar objects without the new keyword complexity.

JavaScript
// Factory function convention: camelCase name
function createUser(name, age, email) {
  return {
    name,       // shorthand for name: name (ES6)
    age,
    email,
    greet() {
      return `Hello, ${this.name}!`;
    },
    isAdult() {
      return this.age >= 18;
    }
  };
}

const alice = createUser('Alice', 30, 'alice@x.com');
const bob   = createUser('Bob', 16, 'bob@x.com');

console.log(alice.greet());    // 'Hello, Alice!'
console.log(bob.isAdult());   // false
Constructor Functions

Constructor functions use new to create objects. By convention, their names start with a capital letter. new automatically creates an empty object, sets this to it, and returns it.

JavaScript
// Constructor function — PascalCase by convention
function Circle(radius) {
  // 'this' refers to the new object being created
  this.radius = radius;
  this.area = function() {
    return Math.PI * this.radius ** 2;
  };
  this.perimeter = function() {
    return 2 * Math.PI * this.radius;
  };
}

const c1 = new Circle(5);
console.log(c1.area());       // 78.53...
console.log(c1.perimeter()); // 31.41...
console.log(c1 instanceof Circle); // true
Value Types vs Reference Types

Primitives = By Value

let a = 10;
let b = a;  // COPY
b = 20;
console.log(a); // 10 — unchanged!
console.log(b); // 20

Objects = By Reference

let x = { n: 10 };
let y = x;   // REFERENCE
y.n = 20;
console.log(x.n); // 20 — x changed!
console.log(y.n); // 20
Cloning Objects
JavaScript — 3 Ways to Clone
const original = { a: 1, b: 2, nested: { c: 3 } };

// 1. Object.assign — shallow clone
const clone1 = Object.assign({}, original);

// 2. Spread operator — shallow clone (modern way)
const clone2 = { ...original };

// 3. JSON trick — deep clone (loses functions/Date)
const deepClone = JSON.parse(JSON.stringify(original));

// Shallow clone SHARES nested objects:
clone2.nested.c = 99;
console.log(original.nested.c); // 99 — both affected!

// Deep clone is truly independent:
deepClone.nested.c = 42;
console.log(original.nested.c); // 99 — unchanged

Adding & Removing Elements
JavaScript
const arr = [1, 2, 3];

// Add to END
arr.push(4, 5);       // [1, 2, 3, 4, 5]

// Add to BEGINNING
arr.unshift(0);       // [0, 1, 2, 3, 4, 5]

// Add in MIDDLE (splice)
arr.splice(2, 0, 99); // at index 2, delete 0, insert 99
// [0, 1, 99, 2, 3, 4, 5]

// Remove from END
arr.pop();             // removes 5, returns 5

// Remove from BEGINNING
arr.shift();           // removes 0, returns 0

// Remove from MIDDLE (splice)
arr.splice(1, 2);     // at index 1, delete 2 items
Finding Elements
JavaScript
const nums = [10, 20, 30, 20];

nums.indexOf(20)          // 1  — first occurrence index
nums.lastIndexOf(20)      // 3  — last occurrence index
nums.includes(30)         // true
nums.indexOf(99)          // -1 — not found

// find — returns first ELEMENT that passes test
const users = [
  { id: 1, name: 'Alice' },
  { id: 2, name: 'Bob'   }
];

const user = users.find(u => u.id === 2);
console.log(user);  // { id: 2, name: 'Bob' }

// findIndex — returns INDEX of first match

Array Higher-Order Methods Must Know

These methods take a callback function and are the backbone of modern JavaScript data processing. They never mutate the original array.

map — transforms each element
const prices = [10, 20, 30];

// Apply 20% discount to each price
const discounted = prices.map(p => p * 0.8);
console.log(discounted); // [8, 16, 24]

// Transform objects
const items = [{ name: 'a', price: 5 }, { name: 'b', price: 10 }];
const names = items.map(item => item.name);
console.log(names); // ['a', 'b']
filter — keeps elements that pass a test
const numbers = [1, 2, 3, 4, 5, 6];

const evens   = numbers.filter(n => n % 2 === 0); // [2, 4, 6]
const bigOnes = numbers.filter(n => n > 3);        // [4, 5, 6]

// Filter out falsy values
const mixed = [0, 1, null, 2, '', 3, undefined];
const truthy = mixed.filter(Boolean); // [1, 2, 3]
reduce — accumulates to a single value
const nums = [1, 2, 3, 4, 5];

// Sum — accumulator starts at 0
const sum = nums.reduce((acc, cur) => acc + cur, 0);
console.log(sum); // 15

// Product
const product = nums.reduce((acc, cur) => acc * cur, 1);
console.log(product); // 120

// Count items with a property
const cart = [{ price: 5 }, { price: 10 }, { price: 3 }];
const total = cart.reduce((acc, item) => acc + item.price, 0);
console.log(total); // 18
every, some, sort, forEach
const ages = [22, 18, 35, 16];

// every — ALL must pass
ages.every(a => a >= 18); // false (16 fails)

// some — at least ONE must pass
ages.some(a => a >= 18);  // true

// sort — sorts in place (mutates original!)
[3, 1, 4, 1, 5].sort();                 // [1, 1, 3, 4, 5] — strings!
[3, 1, 4].sort((a, b) => a - b);     // [1, 3, 4] ascending
[3, 1, 4].sort((a, b) => b - a);     // [4, 3, 1] descending

// forEach — iterate (no return value)
['a', 'b', 'c'].forEach((val, i) => {
  console.log(i, val); // 0 'a', 1 'b', 2 'c'
});
Spread & Rest with Arrays
JavaScript
// Spread — expand array into individual elements
const a = [1, 2, 3];
const b = [4, 5, 6];

const combined = [...a, ...b];  // [1, 2, 3, 4, 5, 6]
const copy = [...a];             // shallow copy

Math.max(...a);                 // 3 — spread as args

// Destructuring with rest
const [first, second, ...rest] = [10, 20, 30, 40];
console.log(first);  // 10
console.log(rest);   // [30, 40]

Declaration vs Expression vs Arrow
JavaScript — 3 Ways to Define Functions
// 1. Function Declaration — hoisted (can call before declaration)
function add(a, b) {
  return a + b;
}

// 2. Function Expression — NOT hoisted
const multiply = function(a, b) {
  return a * b;
};

// 3. Arrow Function (ES6) — concise, no own 'this'

const square   = n => n * n;                // single param — no parens

const getObj   = () => ({ x: 1 });           // return object — wrap in ()

// Multi-line arrow function
const describe = (name, age) => {
  const status = age >= 18 ? 'adult' : 'minor';
  return `${name} is a ${status}`;
};

JavaScript
// Default parameters — used when argument is undefined

function sum(...numbers) {

JavaScript

Getters & Setters
JavaScript
const person = {
  firstName: 'John',
  lastName: 'Doe',

  // getter — access like a property, not a function
  get fullName() {
    return `${this.firstName} ${this.lastName}`;
  },

  // setter — validates / transforms input
  set fullName(value) {
    const [first, last] = value.split(' ');
    this.firstName = first;
    this.lastName = last;
  }
};

console.log(person.fullName);       // 'John Doe'
person.fullName = 'Jane Smith';
console.log(person.firstName);      // 'Jane'
Error Handling: try / catch / finally
JavaScript
function divide(a, b) {
  if (b === 0) throw new Error('Cannot divide by zero');
  return a / b;
}

try {
  const result = divide(10, 0);
  console.log(result);
} catch (error) {
  console.log('Error:', error.message); // 'Cannot divide by zero'
} finally {
  console.log('Always runs'); // cleanup, logging, etc.
}

// Custom error types
function validateAge(age) {
  if (typeof age !== 'number') throw new TypeError('Age must be a number');
  if (age < 0) throw new RangeError('Age cannot be negative');
  return true;
}
The this Keyword
JavaScript
// 'this' refers to the object calling the method
const counter = {
  count: 0,
  increment() {
    this.count++;    // 'this' = counter object
  },
  reset() {
    this.count = 0;
  }
};

// Arrow functions DON'T have their own 'this'

function greet(greeting) {

const boundGreet = greet.bind(user); // new function with this = user

OOP Concepts

Chapter 8 · OOP in JavaScript
The 4 Pillars of OOP

1. Encapsulation

Bundle data + methods together. Hide internal details. Expose only what's needed via a clean interface.

2. Abstraction

Hide implementation complexity. Expose a simple interface. Users don't need to know HOW it works.

3. Inheritance

Child classes inherit properties and methods from parent classes. Promotes code reuse and reduces duplication.

4. Polymorphism

Same interface, different implementation. Objects of different types respond to the same method call differently.

Chapter 9 · OOP in JavaScript
Private Properties (Closures)

JavaScript didn't have true private fields until ES2022. The classic approach uses closures — variables in the constructor function scope that aren't directly accessible from outside.

JavaScript — Closure-based Privacy
function BankAccount(initialBalance) {
  // 'balance' is private — in function scope
  let balance = initialBalance;
  let transactions = [];

  // Public interface (returned object)
  this.deposit = (amount) => {
    if (amount <= 0) throw new Error('Amount must be positive');
    balance += amount;
    transactions.push({ type: 'deposit', amount });
  };

  this.withdraw = (amount) => {
    if (amount > balance) throw new Error('Insufficient funds');
    balance -= amount;
  };

  // Getter — read-only access to private data
  Object.defineProperty(this, 'balance', {
    get: () => balance
  });
}

const account = new BankAccount(100);
account.deposit(50);
console.log(account.balance);  // 150
// account.balance = 9999; ← Won't work — getter only

Prototypes & Prototypical Inheritance

Chapter 10 · OOP in JavaScript
The Prototype Chain

Every JavaScript object has a hidden [[Prototype]] link. When you access a property, JS first looks at the object, then walks up the chain until it finds it or reaches null.

JavaScript
function Animal(name) {
  this.name = name;
}

// Methods on prototype — shared by ALL instances
Animal.prototype.eat = function() {
  return `${this.name} is eating`;
};

Animal.prototype.sleep = function() {
  return `${this.name} is sleeping`;
};

const dog = new Animal('Rex');
console.log(dog.eat());    // 'Rex is eating'

// Property lookup chain:
// dog → Animal.prototype → Object.prototype → null
console.log(dog.toString()); // from Object.prototype

// Own property vs prototype property
console.log(dog.hasOwnProperty('name')); // true (own)
console.log(dog.hasOwnProperty('eat'));  // false (prototype)
Property Descriptors
JavaScript
const obj = { name: 'Alice' };

// View a property's descriptor
Object.getOwnPropertyDescriptor(obj, 'name');
// { value: 'Alice', writable: true, enumerable: true, configurable: true }

// Define a property with specific attributes
Object.defineProperty(obj, 'id', {
  value: 42,
  writable: false,      // cannot be changed
  enumerable: false,   // won't show in for...in
  configurable: false  // cannot be deleted
});

obj.id = 99;              // silently fails (or error in strict mode)
console.log(obj.id);    // 42 — unchanged
Iterating Instance vs Prototype Members
JavaScript
function Circle(r) { this.radius = r; }
Circle.prototype.area = function() { return Math.PI * this.radius ** 2; };

const c = new Circle(5);

Prototypical Inheritance

Chapter 11 · OOP in JavaScript
Creating Custom Inheritance
JavaScript — Manual Prototype Chain
// PARENT
function Shape(color) {
  this.color = color;
}
Shape.prototype.describe = function() {
  return `A ${this.color} shape`;
};

// CHILD
function Circle(color, radius) {
  // Step 1: Call parent constructor
  Shape.call(this, color);
  this.radius = radius;
}

// Step 2: Set up prototype chain
Circle.prototype = Object.create(Shape.prototype);

// Step 3: Fix the constructor reference
Circle.prototype.constructor = Circle;

// Step 4: Add child methods
Circle.prototype.area = function() {
  return Math.PI * this.radius ** 2;
};

const c = new Circle('red', 5);
console.log(c.describe());  // 'A red shape' (inherited)
console.log(c.area());      // 78.53...
console.log(c instanceof Circle); // true
console.log(c instanceof Shape);  // true
Method Overriding & Polymorphism
JavaScript
function Animal(name) { this.name = name; }
Animal.prototype.speak = function() {
  return `${this.name} makes a sound`;
};

function Dog(name) { Animal.call(this, name); }
Dog.prototype = Object.create(Animal.prototype);
Dog.prototype.constructor = Dog;
Dog.prototype.speak = function() {   // OVERRIDE
  return `${this.name} barks!`;
};

function Cat(name) { Animal.call(this, name); }
Cat.prototype = Object.create(Animal.prototype);
Cat.prototype.constructor = Cat;
Cat.prototype.speak = function() {   // OVERRIDE
  return `${this.name} meows!`;
};

// Polymorphism — same method, different behavior
const animals = [new Dog('Rex'), new Cat('Whiskers'), new Animal('Generic')];
animals.forEach(a => console.log(a.speak()));
// 'Rex barks!'
// 'Whiskers meows!'
// 'Generic makes a sound'

Mixins let you add abilities to objects without deep inheritance chains. Great when multiple "traits" are needed.

ES6 Classes

Chapter 12 · OOP in JavaScript
Classes are Syntactic Sugar ES6 classes don't introduce a new inheritance model — they're cleaner syntax over the same prototype-based system. Under the hood, it's still prototypes.
Class Syntax
JavaScript — ES6 Class
class Person {
  // Private field (ES2022)
  #age;

  // constructor — called with 'new'
  constructor(name, age) {
    this.name = name;
    this.#age = age;
  }

  // Instance method
  greet() {
    return `Hi, I'm ${this.name}`;
  }

  // Getter
  get age() { return this.#age; }

  // Setter with validation
  set age(val) {
    if (val < 0) throw new RangeError('Age must be positive');
    this.#age = val;
  }

  // Static method — called on class, not instance
  static compare(p1, p2) {
    return p1.#age - p2.#age;
  }

  toString() {
    return `Person(${this.name}, ${this.#age})`;
  }
}

const alice = new Person('Alice', 30);
const bob   = new Person('Bob', 25);

console.log(alice.greet());           // 'Hi, I'm Alice'
console.log(alice.age);               // 30
alice.age = 31;                       // setter
console.log(Person.compare(alice, bob)); // 6 (static)
Class Inheritance with extends & super
JavaScript — Class Inheritance
class Animal {
  constructor(name, sound) {
    this.name = name;
    this.sound = sound;
  }

  speak() {
    return `${this.name} says ${this.sound}`;
  }
}

class Dog extends Animal {
  constructor(name, breed) {
    super(name, 'Woof');  // MUST call super first
    this.breed = breed;

  // Method override
  speak() {
    return `${super.speak()} (${this.breed})`;
  }

  fetch(item) {
    return `${this.name} fetches the ${item}!`;
  }
}

class GuideDog extends Dog {
  constructor(name, breed, owner) {
    super(name, breed);
    this.owner = owner;
  }
}

console.log(d.fetch('ball')); // 'Rex fetches the ball!'

const g = new GuideDog('Buddy', 'Golden', 'John');
console.log(g.speak());  // 'Buddy says Woof (Golden)'
console.log(g instanceof Animal); // true
console.log(g instanceof Dog);    // true

JavaScript — ES6 Class with all features
class Vehicle {
  #speed = 0;
  static count = 0;

  constructor(make, model, maxSpeed) {
    this.make = make;
    this.model = model;
    this.maxSpeed = maxSpeed;
    Vehicle.count++;
  }

  accelerate(amount) {
    this.#speed = Math.min(this.#speed + amount, this.maxSpeed);
  }

  brake(amount) {
    this.#speed = Math.max(this.#speed - amount, 0);
  }

  get speed() { return this.#speed; }

  toString() {
    return `${this.make} ${this.model} @ ${this.#speed}km/h`;
  }

  static getCount() { return Vehicle.count; }
}

class ElectricCar extends Vehicle {
  #battery;

  constructor(make, model, maxSpeed, batteryKwh) {
    super(make, model, maxSpeed);
    this.#battery = batteryKwh;
  }

  charge() { return `${this.make} charging to 100%`; }

  toString() {
    return `⚡ ${super.toString()} [${this.#battery}kWh]`;
  }
}

const tesla = new ElectricCar('Tesla', 'Model 3', 250, 75);
tesla.accelerate(100);
tesla.accelerate(80);
console.log(tesla.toString());  // ⚡ Tesla Model 3 @ 180km/h [75kWh]
