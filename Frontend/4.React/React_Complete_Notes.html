Part 1 — Getting Started

What is React?

React Overview Core Concept

React is an open-source JavaScript library developed by Facebook (Meta) for building fast, interactive user interfaces — particularly for Single-Page Applications (SPAs). Released in 2013, it became the most popular frontend library in the world.

Core Philosophy

Component-Based — UIs broken into reusable, self-contained pieces.

Declarative — Describe what the UI looks like; React handles DOM updates.

Unidirectional Data Flow — Data flows from parent → child via props.

Virtual DOM — Lightweight copy of DOM to minimize expensive real DOM ops.

React vs Traditional DOM

Traditional (jQuery)
// Manual DOM update
document.getElementById('count')
  .innerText = count + 1;
React Way
function Counter() {
  const [count, setCount] = useState(0);
  return <button onClick={() => setCount(count+1)}>
    {count}
  </button>;
}
Key Mental Model In React, your UI is a function of state: UI = f(state). When state changes, React re-renders the component automatically. You never manually touch the DOM.

Creating Components

Functional Components Modern Standard

A component is a JavaScript function that returns JSX (JavaScript XML) — a syntax extension that looks like HTML but is actually JavaScript. Components are the building blocks of every React application.

Arrow Function Style
const Greeting = () => {
  return <h1>Hello, World!</h1>;
};

export default Greeting;
Named Function Style
function Greeting() {
  return <h1>Hello, World!</h1>;
}

export default Greeting;

JSX — Real-World Component
function UserCard({ name, age }) {
  const isAdult = age >= 18;

  return (
    <div className="card" style={{ padding: '1rem' }}>
      <h2>{name}</h2>
      <p>Age: {age}</p>
      <span>{isAdult ? 'Adult' : 'Minor'}</span>

Fragments

React requires a single root element per component. Fragments let you group elements without adding an extra DOM node — keeping your HTML clean and avoiding unwanted wrappers.

❌ Error — Two Root Elements
return (
  <h1>Title</h1>
  <p>Paragraph</p>
  // ❌ SyntaxError!
);
✅ Fragment — Short Syntax
return (
  <>
    <h1>Title</h1>
    <p>Paragraph</p>
  </>
);
// No extra DOM node added ✅
React.Fragment — Explicit (allows key prop)
return (
  <React.Fragment>
    <h1>Title</h1>
    <p>Paragraph</p>
  </React.Fragment>
);
// Use explicit form when you need the key prop (e.g. in lists)
Rendering Lists

Use JavaScript's .map() method to transform arrays of data into lists of JSX elements. Every item in a list must have a unique key prop to help React identify which items changed, were added, or removed.

Simple List with .map()
const fruits = ['Apple', 'Banana', 'Cherry'];

function FruitList() {
  return (
    <ul>
      {fruits.map((fruit) => (
        <li key={fruit}>{fruit}</li>   // use unique value, NOT index
      ))}
    </ul>
  );
}
Real-World — List of Objects
const users = [
  { id: 1, name: 'Alice', role: 'Admin' },
  { id: 2, name: 'Bob',   role: 'User'  },
];

function UserList() {
  return (
    <ul>
      {users.map(user => (
        <li key={user.id}>
          {user.name} — {user.role}
        </li>
      ))}
    </ul>
  );
}
Why Keys Matter Keys help React reconcile the virtual DOM efficiently. Without keys, React may re-render all items instead of just the changed ones. Never use array index as key if the list can be reordered or filtered — it breaks React's diffing algorithm.
Conditional Rendering

React lets you conditionally render JSX using standard JavaScript constructs. There are three common patterns, each suited for different scenarios.

Pattern 1 — Ternary Operator (show one of two)
function Greeting({ isLoggedIn }) {
  return isLoggedIn
    ? <h1>Welcome back!</h1>
    : <h1>Please log in.</h1>;
}

// Usage:
<Greeting isLoggedIn={true} />  // → "Welcome back!"
<Greeting isLoggedIn={false} /> // → "Please log in."
Pattern 2 — Short-circuit && (show or show nothing)
function Notification({ message }) {
  return (
    <div>
      {message && <p className="alert">{message}</p>}
    </div>
  );
}
// If message is falsy → renders nothing. If truthy → renders <p>
Pattern 3 — Early Return (complex conditions)
function Dashboard({ user }) {
  if (!user)       return <p>Loading...</p>;
  if (user.banned) return <p>Account suspended.</p>;
  return <h1>Welcome, {user.name}!</h1>;
}
GOTCHA — && with falsy values Be careful when the left side of && is 0 or an empty string — React renders those values to the DOM! Use !! to convert to boolean: {!!count && <Badge />}
Handling Events

React uses camelCase event names and passes functions (not strings) as event handlers. React wraps native DOM events in SyntheticEvent objects for cross-browser consistency.

Basic Click Handler
function ClickCounter() {
  const handleClick = (event) => {
    console.log('Button clicked!', event.target);
  };

  return <button onClick={handleClick}>Click Me</button>;
}

// ❌ WRONG — This calls the function immediately on render!
<button onClick={handleClick()}>Oops</button>
Passing Arguments to Event Handlers
function ItemList({ items }) {
  const handleDelete = (id) => {
    console.log('Deleting item:', id);
  };

  return (
    <ul>
      {items.map(item => (
        <li key={item.id}>
          {item.name}
          <button onClick={() => handleDelete(item.id)}>Delete</button>
        </li>
      ))}
    </ul>
  );
}
// Arrow function wrapper () => fn(args) passes arguments correctly

onMouseEnter     ← Mouse enters element
onFocus / onBlur ← Focus gained / lost

Part 3 — Props, State & Communication

Props (Properties)

Props are how you pass data from a parent component to a child component. They are read-only — a child should never modify its own props.

Parent Passes → Child Receives
// Parent — passes various prop types
function App() {
  return (
    <UserCard
      name="Alice"              // string
      age={28}                  // number (use {})
      isAdmin={true}            // boolean
      hobbies={['coding', 'hiking']} // array
    />
  );
}

// Child — destructure from props parameter
function UserCard({ name, age, isAdmin, hobbies }) {
  return (
    <div>
      <h2>{name} {isAdmin && '(Admin)'}</h2>
      <p>Age: {age}</p>
      <ul>{hobbies.map(h => <li key={h}>{h}</li>)}</ul>
    </div>
  );
}
Default Props — Using Default Parameters
function Button({ label = 'Click Me', color = 'blue', onClick }) {
  return (
    <button style={{ backgroundColor: color }} onClick={onClick}>
      {label}
    </button>
  );
}

// Usage with and without props
<Button />                             // → "Click Me", blue
<Button label="Submit" color="green" /> // → "Submit", green
useState Hook Most Used Hook

State is data that can change over time inside a component. When state changes, React re-renders the component. useState returns the current value and a setter function.

Counter — useState Basics
import { useState } from 'react';

function Counter() {
  const [count, setCount] = useState(0); // [currentValue, setter]

  return (
    <div>
      <p>Count: {count}</p>
      <button onClick={() => setCount(count + 1)}>Increment</button>
      <button onClick={() => setCount(count - 1)}>Decrement</button>
      <button onClick={() => setCount(0)}>Reset</button>
    </div>
  );
}
Multiple State Variables — Login Form
function LoginForm() {
  const [email,    setEmail]    = useState('');
  const [password, setPassword] = useState('');
  const [error,    setError]    = useState(null);

  const handleSubmit = () => {
    if (!email) setError('Email is required');
    else setError(null);
  };

  return (
    <div>
      <input value={email}    onChange={e => setEmail(e.target.value)} />
      <input value={password} onChange={e => setPassword(e.target.value)}
             type="password" />
      {error && <p className="error">{error}</p>}
      <button onClick={handleSubmit}>Login</button>
    </div>
  );
}
Critical Rule — Never Mutate State Directly Always call the setter function. React compares references — mutating an object/array directly won't trigger a re-render.
state.name = 'Bob' ❌ — setState({ ...state, name: 'Bob' }) ✅

Part 5 — Managing Component State

useState — Deep Dive

Async State Updates & Updater Function

State updates are asynchronous and batched for performance. React compares new vs old values using Object.is() equality. The updater function form ensures you always get the latest value.

The Stale State Problem & Fix
function Counter() {
  const [count, setCount] = useState(0);

  const handleClick = () => {
    // ❌ WRONG — count is still 0 in both calls, result: count = 1
    setCount(count + 1); // count = 0, so sets to 1
    setCount(count + 1); // count still 0! Sets to 1 again

    // ✅ CORRECT — Updater function always gets current value
    setCount(prev => prev + 1); // prev = 0, result = 1
    setCount(prev => prev + 1); // prev = 1, result = 2 ✅
  };
}
Choosing State Structure

❌ Redundant State

const [firstName, setFirstName] = useState('');
const [lastName,  setLastName]  = useState('');
const [fullName,  setFullName]  = useState('');
// fullName is REDUNDANT — derive it!

✅ Derived Value

const [firstName, setFirstName] = useState('');
const [lastName,  setLastName]  = useState('');
// Computed, not stored in state:
const fullName = `${firstName} ${lastName}`;
State Structure Best Practices Group related state that always changes together · Avoid redundant state — derive values from existing state · Avoid deeply nested state — flatten it to make updates simpler.
Updating Objects in State

State updates must be immutable. Create a new object by spreading the old one and overriding changed fields rather than mutating the existing object.

Object State — Wrong vs Right
const [user, setUser] = useState({ name: 'Alice', age: 28, city: 'NYC' });

// ❌ WRONG — mutates existing object, React won't re-render!
user.name = 'Bob';
setUser(user); // Same reference → no re-render

// ✅ CORRECT — spread old, override changed field
setUser({ ...user, name: 'Bob' });

// Nested object — spread at each level
const [profile, setProfile] = useState({
  name: 'Alice',
  address: { city: 'NYC', zip: '10001' }
});

setProfile({
  ...profile,
  address: { ...profile.address, city: 'LA' }
});
Updating Arrays in State
Array State — All CRUD Patterns
const [items, setItems] = useState(['a', 'b', 'c']);

// ✅ ADD — spread old array + new item
setItems([...items, 'new item']);
setItems(prev => [...prev, 'new item']); // safer with updater form

// ✅ REMOVE — filter out the item
setItems(items.filter(item => item !== 'b'));
setItems(items.filter((_, i) => i !== 1)); // by index

// ✅ UPDATE — map and replace matching item
setItems(items.map(item => item === 'b' ? 'updated' : item));

// ✅ UPDATE OBJECT IN ARRAY — toggle todo.done
setTodos(todos.map(todo =>
  todo.id === id ? { ...todo, done: !todo.done } : todo
));
Simplify with Immer The immer library lets you write mutating-style code that is automatically converted to safe immutable updates. Install with npm install immer use-immer and use useImmer instead of useState.
Lifting State Up

When two sibling components need to share state, lift the state to their closest common ancestor. The ancestor holds the state and passes it down via props.

Temperature Converter — Classic Lifting Example
// ❌ Each has its own state — they can't sync
function TemperatureInput() {
  const [temp, setTemp] = useState(0); // isolated!
}

// ✅ Parent holds shared state, passes down
function Calculator() {
  const [celsius, setCelsius] = useState(0);
  const fahrenheit = (celsius * 9/5) + 32;

  return (
    <>
      <TemperatureInput
        value={celsius}
        onChange={setCelsius}
        label="Celsius"
      />
      <TemperatureDisplay
        value={fahrenheit}
        label="Fahrenheit"
      />
    </>
  );
}

Part 6 — Building Forms

Controlled vs Uncontrolled Inputs

Form inputs can be controlled (React state is the source of truth) or uncontrolled (DOM manages value via refs). Controlled components are the recommended approach in React.

✅ Controlled Input
function ControlledInput() {
  const [value, setValue] = useState('');

  return (
    <input
      value={value}  // React controls
      onChange={e => setValue(e.target.value)}
      placeholder="Type..."
    />
  );
}
Uncontrolled Input (via ref)
function UncontrolledInput() {
  const inputRef = useRef(null);

  const handleSubmit = () => {
    // Read value only on submit
    console.log(inputRef.current.value);
  };

  return <input ref={inputRef} />;
}
React Hook Form (RHF) Performant

React Hook Form is a performant library for managing forms. It uses uncontrolled inputs internally but provides a clean API. It minimizes re-renders compared to controlled forms — a key performance advantage.

React Hook Form — Signup Form
npm install react-hook-form

import { useForm } from 'react-hook-form';

function SignupForm() {
  const { register, handleSubmit, formState: { errors } } = useForm();

  const onSubmit = (data) => {
    console.log(data); // { username: 'Alice', email: '...' }
  };

  return (
    <form onSubmit={handleSubmit(onSubmit)}>
      <input
        {..}register('username', {
          required: 'Username is required',
          minLength: { value: 3, message: 'Min 3 chars' }
        })}
      />
      {errors.username && <p>{errors.username.message}</p>}

      <input
        type="email"
        {..}register('email', { required: 'Email is required' })}
      />
      {errors.email && <p>{errors.email.message}</p>}

      <button type="submit">Sign Up</button>
    </form>
  );
}
Schema Validation with Zod Type-Safe

Zod is a TypeScript-first schema validation library. Combined with RHF's zodResolver, it provides type-safe form validation with a centralized schema definition.

Zod + React Hook Form — Full Schema Validation
npm install zod @hookform/resolvers

import { z } from 'zod';
import { zodResolver } from '@hookform/resolvers/zod';
import { useForm } from 'react-hook-form';

// 1. Define the schema
const schema = z.object({
  username: z.string().min(3, 'Min 3 characters').max(20),
  email:    z.string().email('Invalid email address'),
  age:      z.number().min(18, 'Must be 18+').max(120),
  password: z.string().min(8, 'Min 8 characters'),
});

// 2. Use schema in useForm
function SignupForm() {
  const { register, handleSubmit, formState: { errors } } = useForm({
    resolver: zodResolver(schema),
  });

  const onSubmit = (data) => console.log(data);

  return (
    <form onSubmit={handleSubmit(onSubmit)}>
      <input {..}register('username')} />
      {errors.username && <p>{errors.username.message}</p>}
      <button type="submit">Register</button>
    </form>
  );
}

Part 7 — Connecting to the Back End

useEffect Hook

useEffect lets you perform side effects in function components — operations that interact with the outside world: fetching data, setting up subscriptions, or manually updating the DOM.

useEffect — Three Forms
import { useEffect } from 'react';

function Component() {
  // 1. Runs after EVERY render (no dependency array)
  useEffect(() => {
    console.log('Rendered!');
  });

  // 2. Runs ONCE after initial render (empty array)
  useEffect(() => {
    console.log('Component mounted!');
  }, []); // ← empty array = run once

  // 3. Runs when 'userId' changes
  useEffect(() => {
    fetchUser(userId);
  }, [userId]); // ← runs whenever userId changes
}
Effect Dependencies

The dependency array controls when the effect runs. Include every reactive value (state, props, or variables from render) that the effect uses inside its body.

Fetching Data When userId Changes
function UserProfile({ userId }) {
  const [user, setUser] = useState(null);

  useEffect(() => {
    fetch(`/api/users/${userId}`)
      .then(res => res.json())
      .then(data => setUser(data));
  }, [userId]); // ✅ userId used inside, so include it

  return user ? <h1>{user.name}</h1> : <p>Loading...</p>;
}
ESLint Rule: react-hooks/exhaustive-deps This ESLint rule warns if you forget to include a value in the dependency array. Always follow its recommendations to avoid stale closure bugs — where your effect reads outdated values.
Effect Cleanup

Effects can return a cleanup function that React calls before running the effect again or when the component unmounts. Use this to cancel timers, abort fetch requests, or remove event listeners.

Timer Cleanup
useEffect(() => {
  const timer = setInterval(() => {
    setCount(prev => prev + 1);
  }, 1000);

  // Cleanup — stop timer on unmount
  return () => clearInterval(timer);
}, []);
Event Listener Cleanup
useEffect(() => {
  const handleResize = () =>
    setWidth(window.innerWidth);

  window.addEventListener(
    'resize', handleResize
  );

  return () => window.removeEventListener(
    'resize', handleResize
  );
}, []);
Fetching Data with Axios
Axios — Data Fetching with Loading & Error
npm install axios

import axios from 'axios';
import { useState, useEffect } from 'react';

function PostList() {
  const [posts,   setPosts]   = useState([]);
  const [loading, setLoading] = useState(true);
  const [error,   setError]   = useState(null);

  useEffect(() => {
    axios.get('https://jsonplaceholder.typicode.com/posts')
      .then(res => { setPosts(res.data); setLoading(false); })
      .catch(err => { setError(err.message); setLoading(false); });
  }, []);

  if (loading) return <p>Loading...</p>;
  if (error)   return <p>Error: {error}</p>;

  return (
    <ul>
      {posts.map(post => (
        <li key={post.id}>{post.title}</li>
      ))}
    </ul>
  );
}
Async/Await inside useEffect — Correct Pattern
// ❌ WRONG — useEffect callback cannot be async
useEffect(async () => {
  const data = await fetchData(); // Returns a Promise to React
}, []);

// ✅ CORRECT — define async inside, then call it
useEffect(() => {
  const loadData = async () => {
    try {
      const { data } = await axios.get('/api/users');
      setUsers(data);
    } catch (err) {
      setError(err.message);
    } finally {
      setLoading(false);
    }
  };

  loadData(); // Call it immediately
}, []);
CRUD Operations
Full CRUD — Create, Read, Update, Delete
// CREATE — POST request
const addUser = async (newUser) => {
  const { data } = await axios.post('/api/users', newUser);
  setUsers(prev => [...prev, data]); // optimistic update
};

// UPDATE — PATCH request
const updateUser = async (id, updates) => {
  const { data } = await axios.patch(`/api/users/${id}`, updates);
  setUsers(prev => prev.map(u => u.id === id ? data : u));
};

// DELETE — DELETE request
const deleteUser = async (id) => {
  await axios.delete(`/api/users/${id}`);
  setUsers(prev => prev.filter(u => u.id !== id));
};
Cancelling Requests with AbortController
useEffect(() => {
  const controller = new AbortController();

  axios.get('/api/users', { signal: controller.signal })
    .then(res => setUsers(res.data))
    .catch(err => {
      if (axios.isCancel(err)) return; // ignore cancellation
      setError(err.message);
    });

  return () => controller.abort(); // cleanup on unmount
}, []);
Custom Data Fetching Hook

Extract repeated fetch logic into a custom hook for reusability across components. Custom hooks are functions that start with use and can call other hooks.

useData.js — Reusable Fetch Hook
// src/hooks/useData.js
import { useState, useEffect } from 'react';
import axios from 'axios';

function useData(url) {
  const [data,    setData]    = useState([]);
  const [loading, setLoading] = useState(true);
  const [error,   setError]   = useState(null);

  useEffect(() => {
    const controller = new AbortController();
    setLoading(true);

    axios.get(url, { signal: controller.signal })
      .then(res => { setData(res.data); setLoading(false); })
      .catch(err => {
        if (axios.isCancel(err)) return;
        setError(err.message); setLoading(false);
      });

    return () => controller.abort();
  }, [url]);

  return { data, loading, error };
}

export default useData;
Using useData — 3 Lines Instead of 30!
function PostList() {
  const { data: posts, loading, error } = useData('/api/posts');

  if (loading) return <p>Loading...</p>;
  if (error)   return <p>Error: {error}</p>;

  return (
    <ul>
      {posts.map(p => <li key={p.id}>{p.title}</li>)}
    </ul>
  );
}

⚠️ React Rules to Never Break
1
Hooks at top level onlyNever call Hooks inside loops, conditions, or nested functions.
2
Hooks in React functions onlyNever call Hooks from regular JS functions — only from components or custom hooks.
3
Never mutate state directlyAlways use the setter function and create new objects/arrays.
4
Unique, stable key propsEvery list item needs a unique key. Never use array index as key for reorderable lists.
5
Keep components pureNo side effects during render. Put all side effects in useEffect.
